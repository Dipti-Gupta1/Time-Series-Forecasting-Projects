
# Notebooks

This folder contains the complete notebook-based workflow for the Retail Demand Forecasting project.

Each notebook represents one stage of the end-to-end data science lifecycle.

| Notebook | Description | Status |
|----------|-------------|--------|
| 01_Business_Understanding.ipynb | Define the business problem, objectives, and forecasting goals. | ✅ |
| 01_Data_Understanding.ipynb | Explore all datasets, assess data quality, and understand business context. | ✅ |
| 02_Exploratory_Data_Analysis.ipynb | Analyze trends, seasonality, promotions, holidays, and demand patterns. | ⏳ |
| 03_Feature_Engineering.ipynb | Create time-based, lag, rolling, and external features. | ⏳ |
| 04_Baseline_Model.ipynb | Build and evaluate baseline forecasting models. | ⏳ |
| 05_Classical_Time_Series.ipynb | Implement ARIMA, SARIMA, and Prophet models. | ⏳ |
| 06_Machine_Learning_Models.ipynb | Train tree-based machine learning models. | ⏳ |
| 07_Deep_Learning_Models.ipynb | Build LSTM-based forecasting models. | ⏳ |
| 08_Model_Comparison.ipynb | Compare all models using forecasting metrics. | ⏳ |
| 09_Final_Recommendations.ipynb | Summarize findings and business recommendations. | ⏳ |

---

## Workflow

Business Understanding
→ Data Understanding
→ Exploratory Data Analysis
→ Feature Engineering
→ Baseline Models
→ Classical Time Series
→ Machine Learning
→ Deep Learning
→ Model Comparison
→ Final Recommendations
